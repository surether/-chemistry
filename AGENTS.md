## 기본 원칙

이 저장소에서 Codex는 코드를 수정할 때 반드시 다음 순서를 따른다.

1. 먼저 현재 파일 구조와 관련 코드를 읽는다.
2. 바로 수정하지 말고 작업 계획을 짧게 작성한다.
3. 수정 전 가능한 경우 현재 상태의 실행 또는 테스트를 먼저 수행한다.
4. 오류가 발생하면 traceback, 로그, 실패한 명령어를 근거로 원인을 분석한다.
5. 최소 수정 원칙을 따른다.
6. 수정 후 동일한 명령어를 다시 실행한다.
7. 실패하면 최대 3회까지 `실행 → 오류 분석 → 수정 → 재실행` 루프를 반복한다.
8. 3회 후에도 실패하면 추측으로 계속 수정하지 말고 남은 원인 후보와 필요한 정보를 보고한다.
9. 관련 없는 파일은 수정하지 않는다.
10. 최종 답변에는 다음을 반드시 포함한다.
   - 수정한 파일
   - 실행한 명령어
   - 성공한 검증
   - 실패한 검증
   - 남은 위험

## Python 프로젝트 검증 명령

가능한 경우 다음 순서로 검증한다.

```bash
python -m pytest
python -m py_compile main.py
python main.py --text
```
